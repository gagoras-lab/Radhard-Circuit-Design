//// Radhard Circuit Design ////

The files contained are:

1. s27_mapped.txt: the post-synthesis Verilog file, which contains the circuit we are analyzing.

2. makefile: the file responsible for compiling the code. To compile the code, we type `"make"`, while `"make clean"` deletes the object files produced during compilation (they are not included in the zip; they are generated by the `"make"` command). The file produced by the compiler is the `"executable"`, which is run by the user using the command `"./executable"`.

3. structs.h: the header file that contains the structs used throughout all stages of the project.

4. functions.h: the header file that contains the prototypes of all functions used in the project, along with a brief description of each.

5. circuit_parser.c: the C file where the parsing of the post-synthesis Verilog is handled.

6. simulator.c: the C file where the circuit simulation is performed. The simulation is done recursively from the primary inputs to the outputs of the circuit.

7. truth_table_s27.txt: the file that contains the truth table of the circuit from file 1. It is generated by the simulation and is used as the golden output for comparison in logical masking evaluation.

8. logical_masking.c: the C file that contains the code responsible for the logical masking operation. The evaluation is performed on all intermediate nodes (not primary inputs/outputs), using the simulator developed in the previous stage of the project. For each node, the number of errors found in the faulty truth table is printed, along with the node’s soft error rate. After processing all nodes, the total soft error rate of the circuit is printed.

9. main.c: the C file that contains the main function, responsible for executing all stages of the project. It includes print statements for the start/completion of parsing, and for each node in the circuit it prints its name, its type (input, output, wire), the type of gate(s) connected as inputs, and their output nodes.

10. executable: the executable file produced by our implementation.
